## Risks & Mitigations

### Risk 1: Excel Misused for Recalculation (Violates Read-Only Rule)

**Mitigation:** Lock sheets + no formulas. All exports must be read-only, snapshot-only. No recalculation, no macros, no formulas that recompute pricing. Locked sheets required.

### Risk 2: Data Leakage Through Exports

**Mitigation:** Permission matrix + deny-by-default. Export permissions must be enforced (role-based export access control, hard deny by default). Export permission matrix required.

### Risk 3: Audit Gaps Cause Compliance Issues

**Mitigation:** Mandatory export audit events required. Export audit logging must capture all exports (who/when/what exported). All exports must be auditable.

### Risk 4: Schema Changes Introduced

**Mitigation:** No schema changes allowed. Use existing schema only. Scope guard: no schema changes.

### Risk 5: Logic Duplication in Exports

**Mitigation:** QCD/QCA only rule enforced. All exports must use QCD/QCA only (no duplicate calculators). Exports are read-only snapshots of validated internal state.

### Risk 6: Scope Creep into Approval Workflows or Catalog Export

**Mitigation:** Explicit non-goals defined. Week-10 does not do: approval workflows, catalog data export (unless approved), client-side recompute. Scope guard: export only, no new logic.

### Risk 7: Week-10 Started Before Preconditions Met

**Mitigation:** Hard precondition check. Week-10 CANNOT START unless Week-8.5 Gate = PASS and Week-9 = CLOSED. If any precondition fails, return to originating week, not Week-10.
