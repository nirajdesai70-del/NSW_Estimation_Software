# Changes Directory

This directory tracks planned and implemented changes.

---

## Purpose

- Planned changes documentation
- Change proposals
- Implementation notes
- Change tracking

---

## Structure

```
changes/
├── planned/       # Planned changes
├── implemented/   # Completed changes
└── proposals/     # Change proposals
```

---

## Rules

- All changes must map to baseline
- All changes must be documented
- Original code in `../nish` is never modified
